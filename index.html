      注意！！，git add仅仅是将文件添加到缓存区(stage)，必须要使用git commit 才能将文件添加到分支里面去。

    • git status 是查看此时的工作区情况，告诉此时工作区有哪些变化，eg：有哪些文件是新添加进来的，有哪些文件是发生了修改的，同时这些文件都是在工作区，还没有进入到缓存区，当所有的文件都通过git commit -m “message”提交到了分支之中时，通过log就可以查看添加的文件的指针，方便在后面对需要重新返回的文件进行reset操作。
    • Git管理的是修改，所以每次修改之后，要用add把修改添加到缓存区，添加到缓存区之后，才能利用commit添加到仓库里面，简言之，commit就是你最近一次add或者commit命令之后的缓存区。

  • 工作区，缓存区

    • 工作区：edit了你的文件之后还没add之前
    • add之后，你的文件就保存在了缓存区，等待进一步的commit

    • Git管理的是你的修改，你的每一次的修改，都会保存在记录里面，便于你自己回退

      • 场景1：当你改乱了工作区某个文件的内容，想直接丢弃工作区的修改时，用命令git checkout — file。

      • 场景2：当你不但改乱了工作区某个文件的内容，还添加到了暂存区时，想丢弃修改，分两步，第一步用命令git reset HEAD ，就回到了场景1，第二步按场景1操作。

网络层

发表于 2020-09-27 更新于 2020-10-03 分类于 计算机网络

网络层提供的两种服务

  • 虚电路：一种逻辑上的连接，不是真正的连接，同时这种连接又是必须的，必不可少的
  • 数据报服务： 网络层向上只提供简单的，无连接的，尽最大努力交付的数据报服务，同时，网络层不会提供服务质量的承诺，意味着分组交换的过程之中可能会出错，重复，乱序

    网络协议IP

  • 与IP协议配套使用的协议包括(1，地址解析协议 ARP。2，网际控制报文协议 ICMP。3，网际组管理协议 IGMP)

    分类的IP地址

  • IP地址及其表达方式
  • 常用的三类IP地址和指派范围 在第一个A类的ip地址中，最大可指派的网络数，由于存在网络号全0和全1的情况，所以在计算的时候，要减去全0和全1的两种情况，所以最大可指派的网络数减了一个2，而其他的B类和C类，由于不存在全0和全1的情况，所以不存在减2的操作，所以不会减2。
  • IP地址与硬件地址
    • 物理地址是数据链路层和物理层使用的地址，而IP地址是网络层和以上各层使用的地址，是一种逻辑地址。在发送过程之中，数据从高层到低层，然后才到通信链路上传输，使用IP地址的IP数据报一但交给了数据链路层，就被封装成MAC帧。MAC帧在传送时使用的源地址和目的地址都是硬件地址。（注意PPP帧是在点对点通信时候所采用的一种帧，并且基本是在广域网上面进行通信的，差不多和MAC帧是并列的关系）
    • 需要注意的是，信息的传递过程之中，IP地址中的源地址和目的地址都是没有改变的，改变的是MAC帧的源地址和目的地址

  • IP数据报的格式

    IP数据报首部的固定部分中的各字段的难懂意思

    • 首部长度：一共有4位，每位的单位都是32位字，eg:首部的长度最多达到15(1111)个32位字的长度，总共60字节，所以一般为了减少开销，首部选择5(0101)，这样总共是20字节，不会产生更多的开销
    • 片偏移：较长的分组在分片后，某片在原分组中的相对位置。（简单来说就是起点位置除以8，因为每个分片的长度一定是8字节的整数倍）

  • IP层转发分组的流程

    • （目的网络地址，下一跳地址），所以在路由器中进行传播过程之中，路由器会有一个表，记录下一跳是直接交付还是转到下一个路由器中。
    • 分组转发算法： 简单来说，就是不断的查路由表，以此来找到下一跳的地址，是直接交付，还是下一个路由器的地址。

      划分子网

  • 为啥要划分？
    • 两级IP地址不灵活，IP地址空间的利用率有时很低，给每一个物理网络分配一个网络号表会使路由器变得太大而使网络性能变坏
  • 对外仍是一个网络
  • 同时子网号是从主机号中分出来的。，通过与子网掩码进行想与运算，就知道了子网号是多少。
  • 使用子网时候的分组转发算法 注意：在含有子网掩码的分组转发算法之中，从收到的IP数据报中提取出IP地址，与路由表中每一行都进行想与的操作，只有与这一行所对应的目的网络地址相同，才会进行转发，否则就会从默认路由（如果存在默认路由的话）

    无分类编制CIDR(构造超网)

  • 网络前缀表示IP地址:={<网络前缀>,<主机号>}，子网掩码就是网络前缀的位数，通常在IP地址中，用斜线表示出网络前缀的位数。
  • 最长前缀匹配：从匹配结果之中选择具有最长网络前缀的路由，这样意味着，转发的更加具体，不会那么抽象。（eg：P145）
  • 使用二叉线索查找路由表：找到每一个IP地址的唯一前缀，通过这些唯一前缀，在二叉树中进行查找。

    路由选择协议

    内部网关协议RIP

  • 距离向量算法
  • 从上面的距离向量算法，我们可以看出，RIP协议的特点有
    • 仅和相邻路由器交换信息，不相邻的路由器不交换信息
    • 路由器交换的是当前本路由器所知道的全部信息，即自己现在的路由表。
    • 找固定的时间间隔交换路由信息。
    • RIP存在的一个问题就是当网络出现故障的时候，要经过较长时间才能将此信息传送到所有的路由器。（好消息传递的快，坏消息传递的慢）

      内部网关协议OSPF

  • 最短路径优先：使用了dijkstra提出的最短路径算法SPF
  • OSPF的要点
    • 向本自治系统中所有路由器发送信息
    • 发送的信息就是与本路由器相邻的所有路由器的链路状态，但这只是路由器所知道的部分信息
    • 只有当链路状态发生变化的时候，路由器才向所有路由器用洪泛法发送此信息。不像RIP那样，不管网络拓扑有没有改变，路由器之间都要定期发送信息。

      外部网关协议 BGP

  • 每一个自治系统都配备有一个BGP发言人，同时采用寻找一条比较好的路径（并非最优路径），本自治系统的BGP发言人向其他系统广播到达本自治系统的距离向量。同时BGP发言人除了要运行BGP协议，还要运行本自治系统内的内部网关协议RIP/OPSF

IPv6

从IPv4到IPv6的过度

  • 双协议栈：在完全过渡到IPv6之前，使一些主机或者路由器装有双协议栈，表示他同时有IPv4和IPv6两种IP地址。但需要注意的是，IPv6在传播过程之中。如果经过了只有一个IPv4地址的路由器，最后再恢复到原来的IPv6地址的时候，有些字段是无法恢复的，这种信息的损失是不可避免的。
  • 隧道技术：将IPv6作为IP数据报中的数据部分，封装到IPv4之中，再进行传播。

    IP多播

  • 为了更好地实现一对多的通信，同时，多播地址只能用于目的地址，而不能用于源地址。

    IP多播需要的两种协议

  • IGMP 网际组管理协议
    • 第一阶段：当某台主机加入新的多播组的时候，该主机应向多播组的多播地址发送一个IGMP报文，声明自己要成为改组的成员。
    • 第二阶段：组成员关系是动态的。本地多播路由器要周期性探寻本地局域网上的主机，以便知道这些主机是否还继续是组的成员。
  • 多播路由选择协议
    • 采用三种方法：(1)洪泛和剪枝。(2)隧道技术。(3)基于核心的发现技术

数据链路层

发表于 2020-09-14 更新于 2020-09-27 分类于 计算机网络
  • 数据链路层传输的数据类型—帧

    使用点对点信道的数据链路层(主要针对的是广域网的通信，此时是ppp帧)

  • 通信时候的步骤
    • 结点A的数据链路层把网络层交下来的IP数据报添加首部和尾部
    • 结点A把封装好的帧发送到结点B的数据链路层
    • 若结点B的数据链路层收到的帧无差错，则从收到的帧中提取IP数据报交给上面的网络层；否则丢弃这个帧
  • 数据链路层中的三个功能
    • 封装成帧，透明传输，差错检测
    • 封装成帧：控制字段SOH放在一帧的最前面，表示帧的首部开始，另一控制字符EOF表示帧的结束
    • 透明传输：某一些实际存在的事物看起来却好像没存在一样，ECS，这个转义字符，把数据中可能出现的标识符，全部转换为非标识符，防止出现数据帧的异常开始或者结束
    • 差错检测（循环冗余检验CRC）；采用模2运算，首先规定除数的位数为n+1，所要传送的数据帧，除以这个除数，得到n位的余数（作为FCS），再把这冗余的n位余数，添加到所要传送的数据帧的末尾，这样在接受过程中，用这个新得到的数据帧除以除数，若是得到的余数为0，那么这个传送的数据帧就是没有问题的，没有错误的，否则就是有问题的。
      • 无差错接受：凡是接收端数据链路层接受的帧，我们都能以非常接近于1的概率认为这些帧在传输过程中没有产生差错，可以近似地表述为：凡是接收端数据链路层接受的帧是无差错的

        点对点协议(PPP)

        ppp协议应满足的需求

  • 简单，封装成帧，透明性，多种网络协议，多种类型链路，差错控制，检测连接状态，最大传送单元，网络层地址协商，数据压缩协商

    ppp协议的组成

    (1) 一个将IP数据段封装到串行链路的方法
    (2) 一个用来建立，配置和测试数据链路连接的链路控制协议LCP
    (3) 一套网络控制协议NCP
  • 各个字段的意义
  • 字节填充
    • 异步传输时候，把转义字符定义为0x7D(0111 1101)，并使用字节填充
      (1) 把信息字段中出现的每一个0x7E 字节转变为2字节序列(0x7D,0x5E)
      (2) 若信息字段中出现一个0x7D 的字节(也就是出现了和转义字符一样的比特组合)，那么僵0x7D 转变为2字节序列(07xD,0x5D)
      ……
  • 零比特填充
    • 只要出现了连续的5个1，那么就立即填入一个0，这样可以保证在信息字段中不会出现6个连续的1
    • 接收端在收到一个帧时，先找到标志字段F以确定一个帧的边界，接着再用硬件对比特流进行扫描，当出现连续的5个1时候，删除后面的那个0，还原原来的比特流

      使用广播信道的数据链路层(主要针对的是局域网的通信，此时的是MAC帧)

  • 共享信道技术的两种方法
    • 静态划分信道：频分，时分，波分，码分
    • 动态媒体接入控制：随机接入（可以随机地发送消息），受控接入（用户不能随机地发送信息而必须服从一定的控制）
  • 以太网的两个标准
    • DIX：以太网的第一个规约
    • IEEE 802.3：第一个IEEE的以太网标准
  • 数据链路层被划分为两个子层：逻辑链路控制（LLC），媒体接入控制（MAC），无论采用何种传输媒体和MAC子层的局域网，对LLC来说都是透明的。
  • CAMA/CD协议
  • 协议要点：多点接入，载波监听，碰撞检测
    • 多点接入：计算机以多点接入的方式连接在一条总线上
    • 载波监听：每个站都在不停地检测信道，看信道是否被占用
    • 碰撞检测：边检测边监听，适配器边发送数据边检测信道上的信号电压的变化情况，以便判断自己在发送数据时其他站是否也在发送数据。
  • 由于一个站不可能同时进行发送和接收，因此使用CSMA/CD协议的以太网只能进行双向交替通信。
  • A发送数据之后，要想检测是否发生了碰撞，那么这个从发送数据到知道了发生碰撞这个时间，最多是两倍的总线端到端的传播时延
  • 截止二进制指数退避算法 注意在碰撞次数高于10次以后，k的值就不再变化，就为10，但是，碰撞次数是在增加的。
  • 争用期：以太网端到端的往返时间$2t$，争用期又被称为碰撞窗口，经过争用期这段时间还没有检测到碰撞，才能肯定这次发送不会发生碰撞
  • 以太网在发送数据时候，若在争用期（一共发送64字节）没有发生碰撞，那么后续发送的数据都不会发生碰撞，（检测到碰撞一定会立即停止发送数据，那么发送的长度一定小于64字节，所以凡是小于64字节的都是由于冲突而异常中止的无效帧）
  • 强化碰撞：当发送数据的站发现产生了碰撞时，除了立即停止发送数据之外，还会继续发送32bit/48bit的认为干扰信号，以便让所有的人知道发生了碰撞。

计算机系统的层次结构

  • 具有多层结构的计算机系统

    计算机的基本组成

  • 冯诺依曼计算机结构—以运算器为中心
  • 以存储器为中心的计算机结构
    • 运算器：完成算术运算和逻辑运算，并将运算的中间结果暂存在运算器内
    • 存储器：存放数据和程序
    • 控制器：控制，指挥程序和数据的输入，运行以及处理运算结果
    • 输入设备：用来将人们熟悉的信息形式转换为机器能识别的信息形式，eg：键盘，鼠标
    • 输出设备：可将计算机的运算结果转换为人们熟悉的信息形式，eg：打印机输出，显示器输出
  • 现代计算机的组成框图
  • 主存储器：包括存储体M，各种逻辑部件及控制电路。主存的工作方式是按存储单元的地址号来实现对存储字各位的存（写入），取（读出）
  • 运算器：存放操作数的类型
  • 控制器：计算机的神经中枢，指挥各部件自动协调地工作。
  • I/O：I/O子系统包括各种I/O设备及其相应的接口，每一种I/O设备都由I/O接口与主机联系，它接收CU发出的各种控制命令，并完成相应的操作。
