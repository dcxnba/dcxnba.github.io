题目描述

荷兰旗问题。 按照0,1,2的顺序对数组中的元素进行排序

解法

按照题目中的要求，需要通过原地算法，在原来的数组的基础上进行排序，同时有着重复的数字，那么可以利用指针的方法，通过设置三个指针，$P_0,P_2$代表数组中元素0的右边界，和元素2的左边界，$cur$表示当前元素

那么怎样进行排序呢？
判断$cur$处的元素是多少，如果是0，那么就和$P_0$出的元素交换，如果是2，那么就和$P_2$处的元素进行交换这样$P_0,P_2$指针就把元素0和2分割了开来

不基于模型的控制

发表于 2020-07-22 分类于 Reinforcement Learning

什么是控制？

上一节中的预测，是在给定策略$\pi$下计算状态价值函数或行为价值函数，如何计算呢？无非就是通过$MC$和$TD$方法，进行计算，那么我们就要考虑呢，既然计算出来了，我们怎么通过个体的学习优化价值函数？同时改善策略，以最大化累积奖励的过程？那么，这一过程就叫做控制。

价值函数的表达式

$\pi ‘(s)=argmax_{a\in A} (R_{s}^a+P_{ss’}^aV(s’))$ 从这个式子之中不难看出，我们无法直接通过最大化价值函数的形式，来更新策略，由于状态转移概率$P$在不基于模型的状态下无法直接得出，所以我们必须采取其他办法

不完全的贪婪策略

回顾在不基于模型的预测一节说的内容，我们通过几次采样的差值，来更新价值函数，而每一次对该状态的访问，又都是基于同一策略或者是均方策略，如果是完全贪婪的情况下，容易出现信息不完全，不对称的情况，也会出现采样不完全的情况，那么我们就想可以不可以不那么完全，至少有一些状态还是有机会可以访问，所以就有
$\pi(a|s)=\epsilon /m+1-\epsilon $ 如果$a=argmax_{a\in A}Q(s,a)$
$\pi(a|s)=\epsilon /m$ 其他状况
其中Q是行为价值函数

Sarsa 算法

  • $S_1 A_1 R_2 S_2 A_2..$是不是很熟悉，所以就把这一序列的名字，叫做Sarsa算法
  • 那么你就有疑问了，这和$MC$方法有区别吗？$MC$方法是以多次访问之后价值函数的差进行更新，你就换成了行为函数？这当然有区别，Sarsa在单个状态序列内的每一个时间步，在状态$S$下采取一个行为$A$到达状态$S’$后都要更新状态行为对$(S,A)$的价值$Q(S,A)$这一过程就是采用不完全的贪婪

    Q学习

    策略$\pi$是一个完全贪婪策略。
  • 对比Sarsa和Q学习可以发现，Q的策略偏向于激进，而Sarsa偏向于保守
  • reference 《叶强-强化学习笔记》

  • 有模型和无模型
    在这之前的强化学习问题，均是基于马尔科夫链的预测和策略选择，而$MDP$在现实生活之中不那么容易通过建模，把实际的问题建立成$MDP$，一些情况下利用函数去拟合具体的问题显得格外困难，于是就有了无模型的预测
  • MC学习
    我们知道在一个问题之中，总是有一个基于特定策略$\pi$的一个状态序列:$S_1,A_1,R_2,S_2,A_2……S_t,A_t,R_{t+1}…..S_k$~$\pi$ 在t时刻的收获可以表示为所以就有在某一状态s的价值为:上一节之中，通过一个迭代式更新价值函数，在$MDP$的情况下，有着完整的状态转移矩阵，所以更新时候可以借助贝尔曼方程，结合贪婪的策略对既定策略进行修正。而在无模型的情况下，我们不知道状态转移矩阵，并且也存在着某一状态被多次访问的问题，如何更新价值函数就显得有点棘手。

  • TD 学习

  • 策略评估和策略迭代
    (1)策略评估: 计算给定策略下状态价值函数的过程，通过上节的贝尔曼方程，可以从这个方程中发现，在策略$\pi$下，通过后续的状态计算出当前状态的价值，就是在$MDP$链之中的策略评估，而这个$\pi$可以是均一策略也可以是$greedy$策略
    (2)策略迭代 假设根据贪婪策略的形式来更新初始的策略，那么有
    所以在更新的时候，都是朝着行为价值函数的最大化的情况下进行更新，由此产生一个迭代的过程，贪婪策略对问题中的初始策略进行更新，使得策略的迭代朝着好的方向进行

    通过贝尔曼方程，我们能够知道此时此刻的价值，可以通过后续的价值进行求解，那么就有
    根据最优化和价值最大化的原则，那么就有价值函数的迭代过程之中都是靠着$max$的原则进行

马尔科夫过程

发表于 2020-07-19 更新于 2020-07-20 分类于 Reinforcement Learning
  • 马尔科夫过程中的一些变量
    $S$:有限的状态集合
    $A$:有限的动作集合
    $P$:集合基于行为的状态转移概率矩阵: $P_{ss’}^a=E[R_{t+1}|S_t=s,A_t=a]$
    $R$:基于状态和行为的奖励函数: $R^a_s=E[R_{t+1}|S_t=s,A_t=a]$
    $\gamma$:衰减因子,$\gamma\in[0,1]$
  • 马尔科夫链和马尔科夫过程
    在一个时序过程之中，从某状态采取概率为P的动作，转移到下一个状态，同时产生一个回报R。
    eg:
  • 价值函数和行为价值函数
    (1)价值函数： 啥
    $v(s)=E[G_t|S_t=s]$
    价值函数可以通过期望的定义展开 贝尔曼方程:
    $v(s)=R_s+\gamma_{s’ \in S}\sum P_{ss’}V(s’) $ 个人理解：贝尔曼方程表示在此刻的状态S下的价值函数值为此刻的回报加上往后所有的状态的价值函数
    (2)行为价值函数
    $q_\pi(s,a)=R_s^a+\gamma \sum P_{ss’}^av_\pi(s’) $
    从函数表达式的形式来看就是在状态s下选择动作a之后得到的行为价值函数，由此可见如果对动作a求一个和那么就是价值函数
  • 动作的选择
    贪婪策略:由于要最大化价值策略函数，所以在选择策略的时候就要根据贪婪策略
    $\pi(a|s)=1\ if\ a=argmax_{a\in A}q(s,a)
    \ \ \ else\ 0 $
